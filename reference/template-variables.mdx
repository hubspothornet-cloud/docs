The `AEO-GROWTH-LOOP-TEMPLATE.md` file contains a VARIABLES table at the top listing every placeholder. Fill in every variable before running the loop for the first time — the agent propagates these values into live Shopify content, Knowledge Base entries, and indexed docs articles, so accuracy matters from the start. This reference page documents each variable, the expected format, where to find the value, and common mistakes to avoid.

## Store Identity Variables

<ParamField body="STORE_NAME" type="string" required>
  Your store's display name as shown to customers. Used in generated Q&As, PDP content, and docs articles as the brand attribution.

  **Example:** `SonoHealth`
</ParamField>

<ParamField body="STORE_DOMAIN" type="string" required>
  Your primary store domain without the protocol prefix. Used in structured data and docs links.

  **Example:** `sonohealth.com`
</ParamField>

<ParamField body="SHOPIFY_HANDLE" type="string" required>
  Your store's Shopify myshopify handle — the part before `.myshopify.com`. Used in API calls and connector configuration.

  **Example:** `sonohealth`
</ParamField>

<ParamField body="STORE_RATING" type="string" required>
  Your real aggregate store rating. Must be verifiable. Do not round up or use projected values.

  **Example:** `4.7`
</ParamField>

<ParamField body="REVIEW_COUNT" type="integer" required>
  Your real total review count. Must be verifiable. Update this value periodically as your review count grows.

  **Example:** `1842`
</ParamField>

## Docs Site Variables

<ParamField body="DOCS_DOMAIN" type="string" required>
  Your docs subdomain. Used in structured data `url` fields and internal cross-links generated by the loop.

  **Example:** `docs.sonohealth.com`
</ParamField>

<ParamField body="DOCS_REPO_PATH" type="string" required>
  Absolute local path to your cloned Mintlify repo. The Claude desktop app uses this path to write MDX files during the docs optimization step.

  **Example:** `/Users/you/Documents/GitHub/sonohealth-docs`
</ParamField>

<ParamField body="DOCS_CONFIG_FILE" type="string" required>
  Either `docs.json` (new Mintlify projects) or `mint.json` (older projects). Check your repo root to confirm which one you have — the loop needs this to locate your navigation config.

  **Example:** `docs.json`
</ParamField>

## Product Variables

<ParamField body="CORE_PRODUCT_MON" type="string" required>
  Shopify product handle for the core product focused on **Monday** runs. Find handles in **Shopify Admin → Products → [Product] → SEO section**.

  **Example:** `magnesium-complex-300mg`
</ParamField>

<ParamField body="CORE_PRODUCT_TUE" type="string" required>
  Shopify product handle for the core product focused on **Tuesday** runs.

  **Example:** `blood-pressure-monitor`
</ParamField>

<ParamField body="CORE_PRODUCT_WED" type="string" required>
  Shopify product handle for the core product focused on **Wednesday** runs.

  **Example:** `ekgsmart-monitor`
</ParamField>

<ParamField body="CORE_PRODUCT_THU" type="string" required>
  Shopify product handle for the core product focused on **Thursday** runs.

  **Example:** `pulse-oximeter-pro`
</ParamField>

<ParamField body="ROTATING_PRODUCTS" type="string" required>
  Comma-separated list of Shopify product handles used for the **Friday rotation**. The loop picks from this list each Friday based on analytics.

  **Example:** `blood-pressure-monitor, ekgsmart-monitor, digital-thermometer`
</ParamField>

## Policy & Compliance Variables

<ParamField body="RETURN_POLICY" type="string" required>
  Your exact return policy as stated on your store. Must match your live policy page — this value is written directly into Knowledge Base Q&As and PDP content.

  **Example:** `30-day returns, no questions asked, free return shipping`
</ParamField>

<ParamField body="WARRANTY" type="string" required>
  Your exact warranty terms. Must match what you communicate to customers across all channels.

  **Example:** `1-year manufacturer warranty`
</ParamField>

<Warning>
  Never use aspirational or future-tense values for any variable. The loop propagates these values into live Shopify content, Knowledge Base entries, and indexed docs articles. Inaccurate policy or rating values carry legal and reputational risk.
</Warning>

## Finding Shopify Product Handles

Product handles are available in two places:

**Option 1 — Shopify Admin UI:**
1. Go to **Shopify Admin → Products**.
2. Click any product to open its edit page.
3. Scroll to the **SEO** section at the bottom of the page.
4. The handle appears in the URL slug field.

**Option 2 — Shopify Admin API:**

```http
GET /admin/api/2024-01/products.json?fields=handle,title
```

This returns a JSON array of all products with their handles and titles — useful when you need to audit handles across a large catalog at once.
