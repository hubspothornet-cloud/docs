The `AEO-GROWTH-LOOP-TEMPLATE.md` file contains a VARIABLES table at the top listing every placeholder. This reference page documents each variable, the expected format, where to find the value, and common mistakes to avoid.

## Store Identity Variables

<ParamField path="{{STORE_NAME}}" type="string" required>
  Your store's display name as shown to customers. Used in generated Q&As, PDP content, and docs articles as the brand attribution.

  **Example:** `SonoHealth`
</ParamField>

<ParamField path="{{STORE_DOMAIN}}" type="string" required>
  Your primary store domain without protocol. Used in structured data and docs links.

  **Example:** `sonohealth.com`
</ParamField>

<ParamField path="{{SHOPIFY_HANDLE}}" type="string" required>
  Your store's Shopify myshopify handle — the part before `.myshopify.com`. Used in API calls.

  **Example:** `sonohealth`
</ParamField>

<ParamField path="{{STORE_RATING}}" type="string" required>
  Your real aggregate store rating. Must be verifiable. Do not round up.

  **Example:** `4.7`
</ParamField>

<ParamField path="{{REVIEW_COUNT}}" type="integer" required>
  Your real total review count. Must be verifiable. Update as it grows.

  **Example:** `1842`
</ParamField>

## Docs Site Variables

<ParamField path="{{DOCS_DOMAIN}}" type="string" required>
  Your docs subdomain. Used in structured data `url` fields and internal linking.

  **Example:** `docs.sonohealth.com`
</ParamField>

<ParamField path="{{DOCS_REPO_PATH}}" type="string" required>
  Absolute local path to your cloned Mintlify repo. The Claude desktop app uses this path to write MDX files.

  **Example:** `/Users/you/Documents/GitHub/sonohealth-docs`
</ParamField>

<ParamField path="{{DOCS_CONFIG_FILE}}" type="string" required>
  Either `docs.json` (new Mintlify projects) or `mint.json` (older projects). Check your repo root to confirm which one you have.

  **Example:** `docs.json`
</ParamField>

## Product Variables

<ParamField path="{{CORE_PRODUCT_MON}} – {{CORE_PRODUCT_THU}}" type="string" required>
  Shopify product handles for your four core products, one per weekday (Monday through Thursday). Find handles in **Shopify Admin → Products → [Product] → SEO section**.

  **Example:** `magnesium-complex-300mg`
</ParamField>

<ParamField path="{{ROTATING_PRODUCTS}}" type="string" required>
  Comma-separated list of Shopify handles for Friday rotation.

  **Example:** `blood-pressure-monitor, ekgsmart-monitor, digital-thermometer`
</ParamField>

## Policy & Compliance Variables

<ParamField path="{{RETURN_POLICY}}" type="string" required>
  Your exact return policy as stated on your store. Must match your live policy page.

  **Example:** `30-day returns, no questions asked, free return shipping`
</ParamField>

<ParamField path="{{WARRANTY}}" type="string" required>
  Your exact warranty terms. Must match what you communicate to customers.

  **Example:** `1-year manufacturer warranty`
</ParamField>

<Warning>
  Never use aspirational or future-tense values for any variable. The loop propagates these values into live Shopify content, Knowledge Base entries, and indexed docs articles. Inaccurate policy or rating values carry legal and reputational risk.
</Warning>

## Finding Shopify Product Handles

Product handles are available in two places:

**Option 1 — Shopify Admin UI:**
1. Go to **Shopify Admin → Products**.
2. Click any product to open its edit page.
3. Scroll to the **SEO** section at the bottom of the page.
4. The handle appears in the URL slug field.

**Option 2 — Shopify Admin API:**

```http
GET /admin/api/2024-01/products.json?fields=handle,title
```

This returns a JSON array of all products with their handles and titles — useful when you need to audit handles across a large catalog at once.
